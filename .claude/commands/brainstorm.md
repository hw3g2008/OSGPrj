# /brainstorm 命令

## 用法

```
/brainstorm {模块名或需求描述}
```

## 说明

执行需求头脑风暴，自动迭代正向/反向校验，直到输出完美的需求文档。

## 执行流程

```
1. 触发 Architect Agent
2. 加载 brainstorming Skill
3. 收集相关输入
4. 生成需求初稿
5. 循环正向校验（直到全部 ✅）
6. 循环反向校验（直到全部 ✅）
7. 输出最终需求文档
8. 记录决策到 memory-bank
```

## ⚠️ 执行模式

```
自动迭代执行，不等待用户确认
循环直到所有校验项都是 ✅
```

## 示例

```
/brainstorm 用户登录模块
```

## 输出

需求规格文档，包含：
- 功能需求列表
- 验收标准
- 非功能需求
- 接口定义
