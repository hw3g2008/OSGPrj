# Fix Phase — 诊断（Diagnose）

> **来源**：`debugging` skill 的 Phase 1-3 核心内容提取
>
> **用途**：`fix` skill Phase 1 的执行参考。被 `fix` SKILL.md Phase 1 调用。
>
> **职责边界**：debugging Phase 1-3 负责诊断，**实施修复（写代码）不在此处执行**，由 `fix` Phase 3 执行。

---

## 诊断流程（Phase 1）

### Phase 1.1：复现（Reproduce）

```
[收集错误信息]
    │ - 错误消息
    │ - 堆栈跟踪
    │ - 相关日志
    │
    ▼
[复现问题]
    │ - 记录复现步骤
    │ - 确认可重复
    │
    ▼
[定位范围]
    │ - 确定出错的文件/函数
    │ - 确定出错的行号
```

**硬约束**：
- 禁止跳过复现步骤
- 必须在能稳定复现后，才进入下一步分析

### Phase 1.2：模式分析（Investigate）

```
[检查常见模式]
    │ - 空指针/undefined
    │ - 类型错误
    │ - 边界条件
    │ - 异步问题
    │
    ▼
[查找相似问题]
    │ - 搜索代码库
    │ - 检查历史决策
```

### Phase 1.3：假设测试（Root Cause）

```
[形成假设]
    │ - 基于证据
    │ - 可测试
    │
    ▼
[设计测试]
    │ - 最小化测试
    │ - 隔离变量
    │
    ▼
[执行测试]
    │
    ├── 假设正确 ──→ 输出 Fix Plan
    │
    └── 假设错误 ──→ 新假设（内部重试，上限 3 次）
```

---

## 输出要求

`fix` skill 调用 debugging 时，被调用的 debugging 输出**必须包含**以下四个字段：

| 字段 | 说明 | 约束 |
|------|------|------|
| `root_cause` | 一句话说清根因，带代码位置和证据 | 必须有证据链，不是猜测 |
| `fix_plan` | 最小修改方案（改哪些文件 + 具体改什么 + 为什么最小） | 最小化原则 |
| `impact` | 影响范围（哪些功能可能受影响，如何防止回归） | 必须评估回归风险 |
| `evidence` | 为什么这个假设/方案是正确的证据 | 证据链完整 |

---

## 三次失败规则（调试内部重试）

debugging skill 内部有三次假设级别的重试（同一 hypothesis 的不同验证路径）：

```
失败 1: 分析错误，调整方法
失败 2: 检查假设，扩大搜索范围
失败 3: 停止，请求人工介入或搜索外部资源
```

**注意**：debugging 的三次失败是**调试内部重试**，**不触发 when-stuck**。
- debugging 三次失败 → 返回 `status: escalate` 给调用者（fix）
- fix 的 `attempt_count ≥ 2` 且验证失败 → 触发 when-stuck

## 反合理化表格

| 借口 | 现实检查 |
|------|----------|
| "肯定是这里的问题" | 证据在哪？ |
| "重启应该能解决" | 这不是修复 |
| "环境问题" | 先验证 |
| "之前工作的" | 什么改变了？ |
| "这只是边缘情况" | 能证明不是根因吗？不能 → 禁止 |
| "这次改法不一样" | 根因假设一样吗？一样 → 禁止 |

---

## 红旗 — 停止并遵循流程

- ❌ 随机修改代码希望能工作
- ❌ 没有复现就开始修复
- ❌ 忽略错误消息的内容
- ❌ 同一个修复尝试多次（无新假设）

---

## 输出格式模板

```markdown
## 🔍 诊断报告

### 复现
**步骤**: {用户提供的复现步骤}
**确认**: ✅ 已复现 / ❌ 无法复现

### 根因分析
**假设**: {基于证据的假设}
**验证**: {执行了什么测试}
**确认**: ✅ 假设正确 / ❌ 假设错误

### Fix Plan
**文件**: {需修改的文件路径}
**变更**: {具体修改内容}
**最小化理由**: {为什么不多改}

### 影响范围
{哪些功能可能受影响}
{如何防止回归}

### 经验记录
**根因**: {一句话总结根因}
**预防**: {如何防止同类问题再次发生}
```
