## 环境启动

验收测试基于**真实运行环境**：本地后端 + 远程阿里云 MySQL/Redis + 5个前端 dev server。

⚠️ **本地不启动 Docker**，MySQL/Redis 连远程阿里云共享实例（配置在 `deploy/.env.dev`）。

### ⛔ 数据操作约束

本地**没有** MySQL 和 Redis，它们运行在阿里云远程服务器上。**禁止直接操作远程数据库**：
- **禁止**用 mysql / pymysql / python 等工具直接执行 SQL
- **禁止**用 redis-cli 直接操作 Redis
- 所有数据准备、数据变更**必须通过前端 UI 操作或后端 API 调用**完成
- 如果某步骤无法通过 UI/API 完成，**停下来报告**，由用户决定下一步

### 启动流程

1. 启动后端（唯一入口，参考 `/run-backend-dev` workflow）：
   ```bash
   bash bin/run-backend-dev.sh deploy/.env.dev --check-only   # 预检
   bash bin/run-backend-dev.sh deploy/.env.dev                # 启动
   ```
2. 健康检查：`curl -fsS http://127.0.0.1:28080/actuator/health`
3. 启动 5 个前端 dev server（在 `osg-frontend/` 目录下）：
   ```bash
   pnpm --dir packages/admin dev &
   pnpm --dir packages/mentor dev &
   pnpm --dir packages/lead-mentor dev &
   pnpm --dir packages/assistant dev &
   pnpm --dir packages/student dev &
   ```
4. 等待所有前端启动完毕（各端口可达）

## 执行模式

启动时询问用户选择执行模式：

- **full** — 完整验收（阶段1~6全部执行，约94条用例）
- **smoke** — 冒烟测试（仅阶段0+1+阶段3-冒烟，约7步）
- **l1** — 仅L1单端功能（阶段0+1+2，50条）
- **l2** — L1+L2跨端联动（阶段0+1+2+3，83条）
- **phase N** — 指定从阶段N开始（假设之前阶段的数据已存在）

## 执行规则

1. **严格按用例文档执行** — 不跳步、不简化、不自由发挥
2. **每步截图** — 操作前后各一张，保存到 `yanshou/screenshots/`
3. **失败不阻塞** — 单条失败记录后继续，除非是阶段1数据准备失败
4. **端间切换参照附录E** — 14个切换点表明确了从哪到哪
5. **数据依赖参照附录F** — 确保上游数据存在后再测下游
6. **禁止直接操作远程数据库** — 本地没有 MySQL/Redis，它们在阿里云；所有数据操作必须通过 UI 或后端 API，不得绕过应用层
