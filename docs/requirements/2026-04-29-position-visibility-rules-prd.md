## 一、需求（业务方二轮确认后的终态）

### 1.1 学生端匹配规则
学生端只展示同时满足以下 **三条规则** 的 visible 岗位：
1. **招聘周期**：学生与岗位招聘周期 有任一重合即匹配（集合相交）
2. **求职地区**：学生任一求职大区 与岗位所属大区 一致（只看大区，不看城市）
3. **主攻方向**：学生主攻方向 与岗位 `target_majors` 有任一重合（集合相交）

### 1.2 各端数据范围与筛选项

| 端 | 数据范围 | 筛选项 |
|---|---|---|
| 后台 admin | 全部岗位（含所有状态） | 含状态 + 全维度 |
| 班主任 | 全部 visible 岗位（跨大区、跨班主任） | 现有 5 项（分类/行业/公司/地区/关键词），不加主攻方向 |
| 助教 | 所带学生命中三规则的 visible 岗位 **并集** | 现有 5 项，不加主攻方向 |
| 学生 | 自己命中三规则的 visible 岗位 | 现有维度 |
| 导师 | 不展示岗位列表 | — |

### 1.3 关键定义
- **visible 岗位**：`osg_position.display_status = 'visible'` 且在展示时间区间内、未过期、未下架
- **主攻方向字典**：以 admin 端字典管理中 `osg_major_direction` 为唯一源头（SSOT），所有前端组件通过 `useDictFacade('osg_major_direction')` 动态拉取
- **主攻方向 UI 组件**：统一复用 `osg-frontend/packages/admin/src/views/users/students/components/DirectionCascade.vue`（chip 多选样式）；**该组件当前是硬编码 4 项，需改造为字典化（见 §6.3 #12）**
- **选项内容**：运行时从字典动态加载；业务方在 admin 「字典管理 → 主攻方向」中增删改即生效（如需补 CS / 计算机科学项，在字典中手动添加）
- **学生三个匹配字段**均为多选：`major_direction` / `recruitment_cycle` / `target_region`
- **岗位匹配字段**为新增：`osg_position.target_majors`（多选，必填）
- **学生任一三个匹配字段为空**：列表为空 + 引导补全跳转 `/profile`（沿用现有空态文案）
