# 项目机器真值

本项目的**唯一机器真值入口**是：

- `/.claude/project/config.yaml`
- 执行前置校验：`bin/context-preflight.sh`

这份配置负责定义：

- 环境身份
- 运行时模型
- 真源策略
- 工作流边界

## 1. 规则优先级

优先级固定如下：

1. `/.claude/project/config.yaml`
2. `deploy/runtime-contract.*.yaml`
3. `deploy/.env.*`
4. 说明性文档

说明性文档只负责解释和索引，**不得覆盖** `config.yaml`。

## 2. 当前环境模型

- `dev`
  - 本机运行 backend / frontends
  - 共享远端阿里云 Docker MySQL / Redis
- `test`
  - 阿里云 Docker 运行 backend / frontends
  - 共享远端阿里云 Docker MySQL / Redis
- `prod`
  - 预留结构
  - 以独立 prod runtime contract 为准

## 3. 真源策略

- UI 唯一真源：`osg-spec-docs/source/prototype/*.html`
- `PRD / SRS / MATRIX / UI-VISUAL-CONTRACT / stories / tickets / testing`
  都是派生产物
- 产品确认文档不是第二真源
- 任何 `ui_truth_change=true` 且 `prototype_synced=false` 的决策，都会阻断流程

## 4. 工作流边界

- 正常需求开发：走项目自身工作流
  - `brainstorm -> split-story -> split-ticket -> implement -> verify -> final-gate -> final-closure`
- `superpowers` 只用于：
  - 框架修复
  - 流程修复
  - 真源/guard/runtime 策略修正
- `superpowers` 不得进入任何正常业务交付主链
  - 包括 UI 页面/组件实现与修复
  - 包括 backend/API/数据库实现与修复
  - 包括正常需求开发、测试修复、验证与收口

## 4.1 UI 严格交付策略

- `ui_delivery_policy` 定义 UI 严格收口的机器真值
- UI 修复链路固定为：
  - `single_case_verify`
  - `module_verify`
  - `ui_visual_gate`
  - `final_gate`
  - `final_closure`
- UI 问题不得通过宽阈值、截图旁路或 mask 豁免来过门禁
- 说明性文档只负责解释；真正的 policy 仍以 `config.yaml` 为准

## 5. 关联文件

- 运行契约
  - `deploy/runtime-contract.dev.yaml`
  - `deploy/runtime-contract.test.yaml`
- 环境变量
  - `deploy/.env.dev`
  - `deploy/.env.test`
- 运维说明
  - `deploy/ENV-REQUIREMENTS.md`
  - `deploy/SERVER-DEPLOY-CHECKLIST.md`

如果这些文件与 `config.yaml` 冲突，以 `config.yaml` 为准。

## 6. 新窗口 / 新机器先做什么

任何新的 Codex 窗口、任何新的开发机、以及任何准备执行部署的人，先跑：

```bash
bash bin/context-preflight.sh dev
```

如果目标是远端测试环境，先跑：

```bash
bash bin/context-preflight.sh test --remote-host 47.94.213.128
```

只有 preflight 通过，才允许继续：

- `bash bin/run-backend-dev.sh deploy/.env.dev`
- `bash bin/run-dev-shared.sh`
- `bash bin/deploy-test-remote.sh`
- `bash bin/deploy-server-docker.sh test`
