## 9 · 附带修复：sys_user.user_name VARCHAR(30) → VARCHAR(64)

### 9.1 暴露场景

Step 7 验收 LM 端时发现：通过 admin → 99901 重置密码 / 编辑保存 → backend `MysqlDataTruncation: Data too long for column 'user_name' at row 1`。LM 邮箱 `test-lead-mentor@osg-test.local`（31 chars） > `sys_user.user_name VARCHAR(30)`（ruoyi 上游默认）。

### 9.2 根因

「30 字符」限制散布 4 处：

| # | 位置 | 限制 |
|---|------|------|
| 1 | DB schema `sys_user.user_name VARCHAR(30)` | INSERT MysqlDataTruncation |
| 2 | `SysUser.java:149` `@Size(max=30)` | Bean validation |
| 3 | `SysLoginService.java:166-171` `loginPreCheck` 检查 `username.length() > USERNAME_MAX_LENGTH` → `UserPasswordNotMatchException` | **登录直接 401「用户不存在/密码错误」误导性错误码** |
| 4 | `UserConstants.java:74` `USERNAME_MAX_LENGTH = 30` | 被 #3 + SysRegisterService 引用 |

**实测排查链**：ALTER TABLE 修复 #1 后 INSERT 通过；但登录仍 401，定位到 #3+#4；同步修改后 backend 重启 → LM 登录 200 + token + UI 跳转 /home ✅。

### 9.3 修复变更（5 处）

| 文件 | 改动 |
|------|------|
| `sql/migrations/2026-04-25-sys-user-name-length-64.sql` | 新增 migration，含 `ALTER TABLE sys_user MODIFY COLUMN user_name VARCHAR(64)` |
| `ruoyi-common/src/main/java/com/ruoyi/common/core/domain/entity/SysUser.java:149` | `@Size(max=30→64)` |
| `ruoyi-common/src/main/java/com/ruoyi/common/constant/UserConstants.java:74` | `USERNAME_MAX_LENGTH = 30→64` |
| `ruoyi-framework/src/main/java/com/ruoyi/framework/web/service/SysRegisterService.java:66` | 错误信息 `2到30→2到64` |
| `deploy/mysql-init/00_ry_20250522.sql:47` + `sql/ry_20250522.sql:45` | seed schema `varchar(30→64)` |
