# 学生端岗位真源收口实施计划

> Goal: 按 `docs/plans/2026-03-27-student-position-source-truth-closure-design.md` 的设计，把学生端岗位页收成“正式公共岗位 + 学生自添待审核岗位”两类可解释真源，并与 `我的求职` 正式主状态对齐。

## 0. 执行原则

1. 不恢复旧学生岗位池作为岗位页主真源。
2. 不改变 `我的求职` 作为正式 `osg_job_application` 主视图的定位。
3. 待审核岗位不允许继续伪装成正式可操作岗位。
4. 正式链失败时必须失败，但页面不能出现技术性炸错。

## 1. Workstream B1: 后端岗位列表读链收口

目标：`PositionServiceImpl.selectPositionList` 不再以 `osg_student_job_position` 为主来源。

任务：

1. 审核 `loadVisiblePositions` 当前的三路合并逻辑：
   - 旧学生岗位池 `osg_student_job_position`
   - 学生待审核岗位 `osg_student_position`
   - 正式公共岗位 `osg_position`
2. 改为只保留两类来源：
   - `osg_position`
   - 当前学生的未审核 `osg_student_position`
3. 为岗位条目补齐来源标签或来源判定字段，便于前端决定展示和可操作性。

验收：

- 学生端岗位页列表不再由旧学生岗位池主导
- 任意岗位都能解释是“正式公共岗位”还是“待审核岗位”

## 2. Workstream B2: 公共岗位状态与正式主链对齐

目标：岗位页公共岗位状态尽量由正式 `osg_job_application` 反映。

任务：

1. 梳理当前岗位页依赖的状态字段：
   - `applied`
   - `progressStage`
   - `coachingStatus`
2. 为正式公共岗位引入正式主链状态映射：
   - 无正式申请记录：`未投递`
   - 有正式申请记录：映射正式阶段与辅导状态
3. 保留待审核岗位的独立语义：
   - 仅显示 `待审核`
   - 不映射正式主状态

验收：

- 岗位页公共岗位状态与 `我的求职` 页主状态一致
- 待审核岗位不进入正式求职状态

## 3. Workstream B3: 岗位页操作权限收口

目标：来源决定可操作性，避免待审核岗位伪装成正式岗位。

任务：

1. 明确公共岗位允许的按钮：
   - 投递
   - 取消投递
   - 更新进度
   - 申请辅导
   - 收藏
2. 明确待审核岗位允许的按钮：
   - 查看
   - 收藏（如保留）
3. 禁用待审核岗位上的正式动作：
   - 投递
   - 更新进度
   - 申请辅导
4. 页面展示待审核提示文案。

验收：

- 待审核岗位不再展示正式求职动作
- 来源与可操作性保持一致

## 4. Workstream B4: 前端页面状态与空态调整

目标：学生端页面能准确表达正式状态与受限态，不出现误导。

任务：

1. 调整岗位页卡片/表格状态展示：
   - 公共岗位显示正式状态
   - 待审核岗位显示 `待审核`
2. 调整按钮文案与禁用状态。
3. 保持正式失败时的受控提示，不让未捕获 Promise 泄漏。
4. 保持 `我的求职` 页继续读取正式主链，不引回影子态。

验收：

- 学生端岗位页和 `我的求职` 页状态一致
- 页面不出现假成功或技术性报错

## 5. Workstream B5: 测试补齐

### 5.1 后端测试

补充或调整：

- `PositionServiceImplTest`
  - 岗位列表不再以旧学生岗位池为主来源
  - 公共岗位状态来自正式申请链
  - 待审核岗位不进入正式主状态
  - 待审核岗位不能触发正式动作

### 5.2 前端测试

补充或调整学生端岗位相关测试：

- 岗位页不再依赖旧学生岗位池展示面
- 公共岗位状态与正式主链绑定
- 待审核岗位不出现正式操作按钮
- 页面不出现硬编码假业务值

### 5.3 验收路径

至少验证：

1. 公共岗位
   - 投递成功
   - 岗位页显示已投递
   - `我的求职` 同步出现记录
2. 待审核岗位
   - 手动新增岗位
   - 岗位页显示待审核
   - 不可投递、不可申请辅导、不可更新进度

## 6. 建议实施顺序

1. 先改 `PositionServiceImpl` 列表真源
2. 再补公共岗位状态与正式主链映射
3. 再改前端按钮与状态展示
4. 最后补测试与验收路径

## 7. 完成定义

满足以下条件才算本轮岗位真源收口完成：

1. 学生端岗位列表只保留正式公共岗位和学生自添待审核岗位
2. 公共岗位状态与 `我的求职` 一致
3. 待审核岗位不再具备正式求职动作
4. 页面失败不假成功、不炸错
