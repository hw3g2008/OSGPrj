# HTML Prototype Single Truth Source Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Make HTML prototype the machine-enforced single UI truth source and fail first-order UI derived artifacts when they introduce source-absent pages or surfaces.

**Architecture:** Add a structured truth-source declaration to project config, make source-stage generators consume it, and add a generic derivation consistency guard that compares first-order UI derived artifacts against HTML prototype inventory. Fail early in source-stage, with verify/final only serving as backstops. Downstream stories/tickets/testing remain governed by existing coverage and traceability guards.

**Tech Stack:** Python 3, Bash, YAML, existing workflow-engine guards, project config under `/.claude/project/config.yaml`

## Guard Reuse / Collision Audit

### Reuse Decision

- `ui_visual_contract_guard.py` already validates contract structure and required overlay fields.
- `ui_visual_truth_source_guard.py` validates baseline manifests against contracts.
- Neither guard validates the full derivation chain:
  - HTML prototype inventory -> MATRIX/contract derived artifacts

### New Guard Justification

A new guard is required because the missing responsibility is different:

- not contract schema
- not visual baseline truth source
- but derivation consistency between single truth source and downstream artifacts

### New Guard

- `.claude/skills/workflow-engine/tests/prototype_derivation_consistency_guard.py`

This guard must stay generic and must not collide with existing visual-schema or baseline-manifest guards.

## Source-Stage Integration Path

### Artifact / Contract Generation Order

1. HTML prototype truth source already exists in:
   - `osg-spec-docs/source/prototype/`
2. project-level truth-source declaration is stored in:
   - `.claude/project/config.yaml`
3. source-stage inventory is generated by:
   - `bin/overlay-surface-inventory.py`
4. first-pass surface contract skeleton is generated by:
   - `bin/generate-overlay-surface-skeleton.py`
5. derived-artifact consistency is checked by:
   - `bin/check-skill-artifacts.sh`
   - `prototype_derivation_consistency_guard.py`

No new artifact is allowed to first appear in verify or final-gate.

## Stage-Regression Verification

This work is only complete if the new rule fails at source-stage, not only at final-gate.

Required proof:

1. misconfigured truth-source project config fails in source-stage
2. source-absent surface in `UI-VISUAL-CONTRACT.yaml` fails in source-stage
3. source-absent surface in `MATRIX.md` fails in source-stage
4. verify/final still catch the same mismatch as backstops

### Task 1: Freeze Truth Source Contract In Project Config

**Files:**

- Modify: `.claude/agents/frontend-admin.md`

**Step 1: Add structured truth-source metadata**

Add a machine-readable block under `prd_process`:

```yaml
truth_source:
  type: html_prototype
  root: "${paths.docs.prototypes}"
  single_source_of_truth: true
  derived_artifacts:
    - "${paths.docs.prd}**"
    - "${paths.docs.srs}**"
    - "${paths.tasks.stories}**"
    - "${paths.tasks.tickets}**"
    - "${paths.tasks.root}testing/**"

  require_source_backtrace_for_ui: true
```
